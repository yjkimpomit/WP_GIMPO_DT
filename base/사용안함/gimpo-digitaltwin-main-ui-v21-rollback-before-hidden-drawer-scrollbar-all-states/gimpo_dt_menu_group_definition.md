# 김포 DT 메뉴 그룹별 성격 정의서

## 1. 문서 목적

본 문서는 김포 DT 화면 설계 및 퍼블리싱 반영을 위해 메뉴 그룹별 성격, 하위 메뉴, UI 배치 기준, 3D 연계 방식, Codex 반영 지침을 정리한 문서이다.  
제안요청서 기반 기능을 사용자의 실제 업무 흐름에 맞게 구조화하는 것을 목적으로 한다.

---

## 2. 전체 메뉴 구조

```text
김포 DT
├─ 3D 통합 화면
├─ 설비자료
├─ 안전
├─ AI 현장관제
├─ GENi
└─ AR 현장점검
```

권장 1Depth 메뉴 순서는 다음과 같다.

1. 3D 통합 화면
2. 설비자료
3. 안전
4. AI 현장관제
5. GENi
6. AR 현장점검

3D 통합 화면을 최상위에 배치하는 이유는 김포 DT의 핵심 차별화 요소가 3D 기반 디지털트윈 진입 구조이기 때문이다. GENi는 기존 시스템 연계 성격이 강하므로 독립 메뉴로 유지하되, 3D 설비 선택 후 바로가기 버튼으로도 반복 노출될 수 있다.

---

## 3. 3D 통합 화면

### 3.1 메뉴 성격

3D 통합 화면은 김포 DT의 메인 진입 허브이다.  
사용자가 3D 모델에서 측정설비 또는 주요 설비를 선택하면, 해당 설비와 연결된 태그 기반 정보들이 바로가기 버튼그룹으로 표시되는 구조를 가진다.

일반적인 메뉴 탐색 방식이 아니라, “설비 선택 → 관련 정보 호출”의 컨텍스트 메뉴 구조로 설계한다.

### 3.2 주요 기능

- 3D 설비 모델 탐색
- 태그 기반 설비 선택
- 선택 설비 기준 연계 정보 표시
- 관련 시스템 바로가기 제공
- 설비별 상태, 자료, 점검, 정비 정보 접근

### 3.3 바로가기 버튼그룹 구성

설비 선택 시 표시되는 버튼그룹은 다음 기준으로 구성한다.

| 구분 | 버튼 |
|---|---|
| 기본정보 | 설비정보, TM현황 |
| 도면/영상 | P&ID, CCTV, OWS |
| 정비/점검 | 점검현황, 미결작업, RedTAG |
| 현장연계 | AR 점검마커 |

### 3.4 UI 반영 기준

- 3D 화면은 좌측 메뉴의 첫 번째 항목으로 배치한다.
- 설비 선택 전에는 기본 안내 또는 주요 설비 목록을 제공한다.
- 설비 선택 후에는 우측 패널 또는 하단 패널에 바로가기 버튼그룹을 표시한다.
- 버튼은 기능군별로 시각적 구분이 가능해야 한다.
- GENi, 설비자료, AR 메뉴와 중복되는 기능은 허용한다. 단, 3D에서는 “선택 설비 기준 바로가기”로 동작해야 한다.

## 4. 설비자료 그룹

### 4.1 메뉴 성격

설비자료는 설비와 관련된 정적·시각 자료를 조회하는 메뉴 그룹이다.  
운영자가 현재 상태를 확인하거나, 현장 영상·도면·운전 화면을 참조하는 기능을 묶는다.

업무 액션보다는 조회 중심 성격이 강하다.

### 4.2 하위 메뉴

```text
설비자료
├─ 대시보드
├─ 파노라마
├─ CCTV
├─ P&ID
└─ OWS
```

### 4.3 하위 메뉴 역할

| 메뉴 | 역할 |
|---|---|
| 대시보드 | 설비 상태 및 주요 지표 요약 |
| 파노라마 | 현장 공간 또는 설비 주변 시각 자료 조회 |
| CCTV | 설비 또는 현장 영상 확인 |
| P&ID | 배관 및 계장 도면 조회 |
| OWS | 운전 화면 또는 운전 관련 화면 조회 |

### 4.4 권장 표시 순서

1. 대시보드
2. 파노라마
3. CCTV
4. P&ID
5. OWS

요약 정보에서 시작해 현장 시야, 영상, 도면, 운전 화면으로 정보 밀도가 높아지는 흐름이다.

### 4.5 UI 반영 기준

- 좌측 메뉴에서는 “설비자료” 그룹으로 묶는다.
- 각 하위 메뉴는 아이콘을 구분하되, 과도한 장식보다 조회 기능임을 명확히 표현한다.
- 3D 화면에서 설비를 선택했을 때 설비자료 일부가 바로가기 버튼으로 호출될 수 있다.
- P&ID와 OWS는 도면 또는 운전 화면 성격이 강하므로, 화면 내 확대·축소·이동 UI를 고려한다.

## 5. 안전 그룹

### 5.1 메뉴 성격

안전 그룹은 안전진단, 설비진단, 예측 기반 위험 감지 기능을 묶는 메뉴이다.  
두케어 3종과 표준복합 5종을 포함한다.

서비스명만 노출할 경우 사용자가 기능을 바로 이해하기 어려울 수 있으므로, 메뉴명 또는 보조 설명에 기능 설명을 함께 제공하는 것이 좋다.

### 5.2 하위 메뉴

```text
안전
├─ 두케어
│  ├─ gasPED
│  ├─ DCAT
│  └─ PreVISION
└─ 표준복합
   ├─ 발전량 예측
   ├─ 성능 예측
   ├─ 이상징후 예측
   ├─ 운전상태 분석
   └─ 예방진단
```

### 5.3 두케어 메뉴 역할

| 메뉴 | 권장 보조 설명 |
|---|---|
| gasPED | 가스설비 이상진단 |
| DCAT | 설비상태 진단 |
| PreVISION | 예측 기반 안전진단 |

### 5.4 표준복합 메뉴 역할

| 메뉴 | 역할 |
|---|---|
| 발전량 예측 | 운전 데이터 기반 발전량 예측 |
| 성능 예측 | 설비 성능 저하 또는 효율 변화 예측 |
| 이상징후 예측 | 운전 데이터 기반 이상징후 탐지 |
| 운전상태 분석 | 운전 조건 및 상태 데이터 분석 |
| 예방진단 | 이상 발생 전 예방 관점의 진단 |

### 5.5 UI 반영 기준

- 좌측 메뉴에서는 “안전”을 1Depth로 두고, 두케어와 표준복합을 2Depth 그룹으로 둔다.
- 두케어는 서비스명 유지가 필요하다.
- 표준복합은 업무 기능명이 명확하므로 기능명 중심으로 표시한다.
- 안전 관련 메뉴는 경고, 주의, 정상 상태 등 상태 색상을 사용할 수 있으나, 메뉴 자체의 기본 색상은 과도하게 위험색 중심으로 구성하지 않는다.
- 카드 또는 상세 화면에서는 상태 뱃지, 위험도, 예측 결과, 최근 갱신 시각을 함께 제공하는 구조가 적합하다.

## 6. AI 현장관제 그룹

### 6.1 메뉴 성격

AI 현장관제는 현장 작업자, 건강 상태, 로봇 감지정보를 AI 기반으로 모니터링하는 메뉴 그룹이다.  
단순 AI 기능 모음이 아니라 현장 안전과 감시를 위한 실시간 관제 성격으로 정의한다.

“AI” 단독 명칭보다 “AI 현장관제”가 메뉴 목적을 더 명확하게 전달한다.

### 6.2 하위 메뉴

```text
AI 현장관제
├─ 작업자 위치
├─ 건강 모니터링
└─ 4족 보행로봇
   ├─ 감지정보
   ├─ 순찰이력
   └─ 이상감지 알림
```

### 6.3 하위 메뉴 역할

| 메뉴 | 역할 |
|---|---|
| 작업자 위치 | 현장 작업자의 위치 및 이동 상태 확인 |
| 건강 모니터링 | 작업자의 생체 또는 건강 상태 모니터링 |
| 4족 보행로봇 감지정보 | 로봇이 감지한 현장 정보 확인 |
| 순찰이력 | 로봇의 이동 경로 및 순찰 기록 확인 |
| 이상감지 알림 | 로봇 또는 AI가 감지한 이상 상황 표시 |

### 6.4 UI 반영 기준

- 메뉴명은 “AI 현장관제”를 권장한다.
- 위치, 건강, 로봇 정보는 실시간 또는 준실시간 정보로 인식되므로 갱신 시각 표시가 필요하다.
- 지도, 3D 위치, 현장 평면도, 알림 리스트와 연계될 수 있다.
- 안전과 중복되는 정보가 있더라도, AI 현장관제는 “실시간 현장 감시” 중심으로 구분한다.
- 작업자 건강 정보는 개인정보 민감도가 높으므로 화면 노출 범위와 권한 제어를 고려해야 한다.

## 7. GENi 그룹

### 7.1 메뉴 성격

GENi는 기존 iPOS 또는 GENi 기반 업무정보와 연계되는 메뉴 그룹이다.  
김포 DT에서 새롭게 재해석하기보다 기존 업무 명칭과 구조를 유지하는 것이 사용자의 학습 부담을 줄인다.

3D 화면에서는 선택 설비 기준의 바로가기 버튼으로도 제공될 수 있다.

### 7.2 하위 메뉴

```text
GENi
├─ 설비정보
├─ TM현황
├─ RedTAG
├─ 점검현황
├─ 미결작업
└─ 자재정보
```

### 7.3 하위 메뉴 역할

| 메뉴 | 역할 |
|---|---|
| 설비정보 | 선택 설비의 기본 정보 및 상세 정보 조회 |
| TM현황 | 태그 또는 계측 데이터 기반 현황 조회 |
| RedTAG | 위험·주의·관리 대상 태그 정보 확인 |
| 점검현황 | 설비별 점검 상태 및 이력 확인 |
| 미결작업 | 완료되지 않은 작업 또는 정비 항목 확인 |
| 자재정보 | 설비 관련 자재 및 예비품 정보 조회 |

### 7.4 UI 반영 기준

- 메뉴명은 기존 명칭을 최대한 유지한다.
- RedTAG처럼 업무 시스템에서 이미 인지된 명칭은 그대로 사용한다.
- GENi 메뉴는 좌측 메뉴에서 독립 그룹으로 제공한다.
- 3D 화면에서 설비를 선택하면 GENi의 설비정보, TM현황, 점검현황, RedTAG 등이 바로가기 버튼으로 표시될 수 있다.
- 기존 iPOS와의 호환성을 고려해 메뉴 구조, 명칭, 데이터 표시 방식은 과도하게 변경하지 않는다.

## 8. AR 현장점검 그룹

### 8.1 메뉴 성격

AR 현장점검은 모바일 기기와 AR 점검마커를 활용하여 현장에서 점검 업무를 수행하는 메뉴 그룹이다.  
데스크톱 기반 조회 기능과 달리 현장 수행 업무에 가까우므로 별도 메뉴로 분리한다.

### 8.2 하위 메뉴

```text
AR 현장점검
├─ AR 점검마커
├─ 모바일 현장점검
├─ 점검결과 등록
└─ 점검이력 조회
```

### 8.3 하위 메뉴 역할

| 메뉴 | 역할 |
|---|---|
| AR 점검마커 | 설비별 AR 마커 정보 확인 |
| 모바일 현장점검 | 모바일 기기 기반 점검 수행 |
| 점검결과 등록 | 현장 점검 결과 입력 및 저장 |
| 점검이력 조회 | 과거 점검 내역 확인 |

### 8.4 UI 반영 기준

- 메뉴명은 “AR 현장점검”을 권장한다.
- AR 기능은 데스크톱 메뉴에서 진입하더라도 실제 수행은 모바일 환경을 전제로 한다.
- 3D 화면에서 설비 선택 시 AR 점검마커 바로가기를 제공할 수 있다.
- 점검결과 등록 화면은 현장 사용성을 고려해 입력 항목을 최소화하고, 체크형 UI를 우선 적용한다.
- 점검이력 조회는 GENi의 점검현황과 연계될 수 있으나, AR 메뉴에서는 AR 기반 수행 이력 중심으로 구분한다.

## 9. 메뉴 간 중복 처리 기준

일부 기능은 3D, GENi, 설비자료, AR에서 중복 노출될 수 있다.  
중복 자체가 문제는 아니며, 진입 맥락에 따라 역할을 구분해야 한다.

| 기능 | 3D에서의 역할 | 독립 메뉴에서의 역할 |
|---|---|---|
| 설비정보 | 선택 설비 기준 바로가기 | 전체 설비 정보 조회 |
| TM현황 | 선택 설비 태그 현황 | 태그/계측 데이터 전체 조회 |
| P&ID | 선택 설비 관련 도면 호출 | 도면 목록 및 상세 조회 |
| CCTV | 선택 설비 주변 영상 호출 | CCTV 목록 및 영상 조회 |
| AR 점검마커 | 선택 설비 AR 점검 진입 | AR 점검 업무 전체 관리 |
| 점검현황 | 선택 설비 점검 상태 확인 | 전체 점검 현황 조회 |

### 10.3 퍼블리싱 반영 기준

- 좌측 메뉴는 1Depth 메뉴 6개를 기본으로 한다.
- “안전”, “AI 현장관제”, “GENi”, “AR 현장점검”은 하위 메뉴를 갖는 그룹으로 구현한다.
- “3D 통합 화면”은 일반 목록 메뉴라기보다 메인 허브 화면으로 구현한다.
- 3D 설비 선택 시 바로가기 버튼그룹은 선택 설비 컨텍스트에 따라 노출한다.
- 메뉴명은 본 문서의 권장 명칭을 우선 사용한다.
- 기존 iPOS/GENi와 연결되는 명칭은 과도하게 변경하지 않는다.
- 김포 DT 차별화 요소는 3D, AI 현장관제, AR 현장점검에서 강조한다.

## 11. 제안서 요약 문구

김포 DT의 메뉴 구조는 3D 통합 화면을 중심으로 설비 선택 시 태그 기반 연계 정보를 제공하는 구조로 설계한다. 설비자료, 안전, AI 현장관제, GENi, AR 현장점검을 주요 업무 그룹으로 분리하여 사용자가 설비 상태 확인, 안전진단, 현장 감시, 정비정보 조회, 모바일 점검 업무를 일관된 흐름으로 수행할 수 있도록 구성한다.
