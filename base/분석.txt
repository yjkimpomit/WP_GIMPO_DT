## 제안요청서 분석 반영 디자인 화면 설명

1. **사업 목적 반영**

   * 발전소 기동·정지 절차 평가 업무를 효율적으로 수행할 수 있도록 교육, 평가, 관리 기능을 통합한 화면 구조로 설계하였다.
   * 사용자가 평가 대상, 진행 상태, 학습 필요 항목을 빠르게 파악할 수 있도록 대시보드 중심의 정보 구조를 적용하였다.

2. **업무 프로세스 기반 화면 구성**

   * 평가 준비, 문제 풀이, 제어화면 확인, 답변 입력, 결과 확인까지의 흐름을 실제 업무 절차에 맞춰 구성하였다.
   * 절차 기반 평가가 가능하도록 좌측에는 평가 단계, 중앙에는 제어화면, 우측에는 문제 및 답변 영역을 배치하였다.

3. **현장 제어화면 연계 반영**

   * 실제 발전소 제어화면과 유사한 조작·확인 환경을 제공할 수 있도록 중앙 제어화면 영역을 핵심 화면으로 구성하였다.
   * 설비 객체, Tag 값, 절차별 상태값을 평가 문제와 연계할 수 있는 구조를 고려하였다.

4. **사용자 대시보드 설계**

   * 오늘 수행해야 할 평가, 대기 중인 평가, 취약 절차, 공지사항, 학습 진행 상태를 한 화면에서 확인할 수 있도록 구성하였다.
   * 평가자가 현재 해야 할 업무를 우선 인지할 수 있도록 중요도에 따라 카드형 정보 구조를 적용하였다.

5. **관리자 기능 반영**

   * 평가 운영, 문제 관리, 제어화면 관리, 객체/Tag 관리, 결과 관리 등 관리자 업무를 체계적으로 분리하였다.
   * 평가 생성부터 배정, 진행 현황 확인, 결과 분석까지 관리자가 수행하는 운영 흐름을 기준으로 메뉴를 구성하였다.

6. **평가·학습 연계 구조**

   * 평가 결과를 단순 점수 확인에 그치지 않고 취약 절차 분석과 연계하여 반복 학습이 가능하도록 설계하였다.
   * 사용자의 오답, 미숙 단계, 반복 취약 구간을 시각적으로 확인할 수 있는 분석 영역을 반영하였다.

7. **정보 가독성 및 시각적 위계**

   * 주요 업무 정보는 카드형 UI로 구분하고, 상태값은 색상·배지·진행률을 활용해 직관적으로 표현하였다.
   * 평가 진행 상태, 알림, 경고성 정보는 우선순위를 명확히 하여 사용자가 빠르게 판단할 수 있도록 구성하였다.

8. **접근성과 사용성 고려**

   * 복잡한 평가 업무에서도 사용자가 학습 없이 화면을 이해할 수 있도록 명확한 메뉴명과 업무 중심 문구를 적용하였다.
   * 버튼, 탭, 카드, 상태 표시 요소는 일관된 규칙으로 구성하여 반복 사용 시 인지 부담을 줄였다.

9. **개발 구현 가능성 반영**

   * 화면 구조는 컴포넌트 단위로 분리 가능한 카드, 표, 트리, 탭, 패널 중심으로 설계하였다.
   * 향후 데이터 연동, 객체값 매핑, 평가 결과 집계, 관리자 기능 확장에 대응할 수 있는 구조를 고려하였다.

10. **전체 디자인 방향**

* 발전소 운영 시스템의 전문성과 신뢰성을 유지하면서도, 평가시스템으로서의 사용 편의성을 강화한 업무형 UI로 설계하였다.
* 단순 시각 디자인이 아니라 제안요청서의 기능 요구, 업무 흐름, 운영 관리 체계를 화면 구조에 반영한 디자인이다.
